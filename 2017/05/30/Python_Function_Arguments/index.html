内容：主要是对函数定义时的参数(parameter)以及函数调用时的参数(argument)作一个简单的介绍。Python版本(3.6.1)

arguments和parameters的区别

函数定义中的参数名称是parameters，函数调用时传递给函数的值是arguments；对照我们平常的说法，就是parameters是形参，arguments是实参。

Parameters决定了一个函数在调用时可以接收什么类型的arguments。

argument详细介绍

argument指的是调用function时传递给function(或method)时的值。argument有如下2种类型：

  • keyword argument：调用时前面带有标识符的argument(如，name＝)或者带有**前缀的字典中的argument。例如，下述add()调用过程中的1，2都是keyword argument。

  • positional argument：不是keyword argument的arguemnt就是positional argument。。。Positional argument既可以出现在argument列表的开头，也可以作为带有*前缀的iterable中的元素，当然两种情况也可以同时出现。例如，下述add()调用过程中1，2都是postional argument。

parameter详细介绍

parameter指的是function(或method)定义中被命名的实体，指定了function在调用时可以接收的argument类型。parameter有如下5种类型：

  • positional-or-keyword：表示既可以接收positional argument也可以接收keyword argument，这种类型是parameter的默认类型。例如下述func定义中的foo，bar：

  • positional-only：表示只能接收positional argument。Python并没有对应的语法来定义positional－only parameter。然而有些Python内置function有positional－only parameter(例如 abs())。

  • var-positional：表示可以接收任意positional arguments参数的sequence，当然已经被其他parameters接收的positional arguments除外。可以通过在parameter名称前加上*来定义var-positional parameter。例如下述中的args。

  • var-keyword：表示可以接收任意多的keyword arguments参数，当然已经被其他parameters接收的keyword arguments除外。可以通过在parameter名称前加上**来定义var-keyword parameter。例如上述中的kwargs。

  • keyword-only：表示只能接收keyword argument。在function的parameter列表中，如果有的parameter前有一个唯一的var-positional parameter或者单独的*,那么这些parameter就是keyword-only parameter。例如下述中的kw_only1, kw_only2：

Parameters can specify both optional and required arguments, as well as default values for some optional arguments.
