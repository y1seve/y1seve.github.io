变量

变量声明

在Java中，声明变量时需要指明变量类型。但是Kotlin中，在声明变量时可以忽略变量类型。

声明变量name和country时，忽略了类型；声明变量num时，mi显示地指明了类型。

如果不指明类型，编译器会分析这个声明进而推断出类型。这就叫做类型推断（type inference）。Kotlin中很多地方都存在类型推断。

依照类型推断，那么编译器会推断name和country都为String类型。

如果在声明变量时，没有初始化，那么在声明时必需指明类型。因为没有初始化，编译器没有足够的信息去推断变量类型。

可变和不可变

声明变量时，会用到两个关键字：

  • val（value）引用不可变。用val声明的变量在初始化后不能再被赋值。和Java中的final variable一致。

  • var（variable）引用可变。用var声明的变量，可以在初始化后赋值。和Java中的普通变变量一致。

Kotlin中，应该尽量用val来声明变量，只在需要时，用var声明变量。使用不可变的引用，不可变的变量和没有边际影响的函数可以使编写的代码更加具有函数式的风格。
