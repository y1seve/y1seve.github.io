Manipulating Elements

完整的jQuery操作元素的方法见官方文档。

Getting and Setting Information About Elements

有许多方式去改变一个存在的元素。其中最常用的就是改变元素的inner HTML和属性。jQuery提供了简单，并且跨浏览器的方法来支持这些操作。你可以通过这些方法的getters的形势来获取关于元素的信息。下面是一些方法：

  • .html() - get or set the html contents.
  • .text() - get or set the text contents; HTML will be stripped.
  • .attr() - get or set the value of the provided attribute.
  • .width() - get or set the width in pixels of the first element in the selection as an integer.
  • .height() - get or set the height in pixels of the first element in the selection as an integer.
  • .position() - get an object with position information for the first element in the selection, relative to its first positioned ancestor.This is a getter only.
  • val() - get or set the value of form elements.

改变元素的相关信息是很简单的，但是要记住的是这种改变会影响这次选择到的所有元素。如果你想改变的只是其中的一个元素，则需要在调用该方法的setter之前指明。

Moving, Copying, and Removing Elements

有许多方式在DOM中移动元素，一般有如下两种方式：

  • 相对另一个元素放置被选中的元素。
  • 相对被选中的元素放置某一个元素。

例如，jQuery提供.insertAfter()和.after()方法。.insertAfter()方法是将被选中的元素放置于作为参数的那个元素的后面。.after()方法是将作为参数的那个元素放置于被选中的元素后面。其他的方法都是遵循此规则：.insertBefore() .before() .appendTo() .append() .prependTo() .prepend()

Removing Elements

可以使用.remove()和.detach()方法来从页面中移除元素。当你想从页面中永久移除被选择的元素时可以使用.remove()方法。当调用.remove()方法时，会返回已经移除的所有元素，如果你再次将这些元素再添加到页面中去，那这些元素已经没有移除前关联的数据和事件了。

然而.detach()方法依然会有移除前关联的数据和事件。

.detach()方法在你频繁操作元素的时候特别有用。比如，你可以使用.detach()方法来从页面移除元素，然后代码会继续执行某件任务，当这件任务完成后，你又将先前移除的元素又添加到页面。这就会在维持元素的数据和事件方面减少 “DOM touches”的消耗。

如果你想将某一个元素留在页面，但是移除该元素的内容，那么你可以使用.empty()方法去清除该元素的inner HTML.

Working with Selections

Getters & Setters

一些jQuery方法可以既可以用于赋值给选择的元素，也可以读取该元素的值。当这个方法带参数调用时，就可看作是setter；当这个方法空参数调用时，就可看作时getter。Setter操作影响这次选择的所有元素，Getter操作只会读取这次选择的元素中的第一个元素对应的值。（除了.text()例外，该方法会返回选择到的所有元素对应的值）。

Setters方法返回一个jQuery对象，可以允许你继续这次选择上调用jQuery方法。Getters返回的事你想返回的内容，所以你不能继续在Getters返回的值上调用jQuery方法。

Chaining

如果你在一次选择上调用了jQuery方法后，返回的是一个jQuery对象，那么你可以继续在返回的对象上再调用jQuery方法，而不用通过分号来打断调用。这种方式叫做chaining.

将上述链式调用写成多行会使得代码可读性更高

jQuery也提供.end()方法终结掉最近的过滤操作，相当于你可以在调用链中间改变选择到的元素。

Chaining是一种极其强大的方式，该方式因为在jQuery中的使用而在其他的js库中也流行起来。但是，该方式使用的时候要非常小心，大范围的chaining会使得代码相当难地去修改和调试。并没有什么规则表明一个调用链是否过长，知道这点就行。

判断是否选择到了元素的方式是判断.length属性，如果没有选择到元素，该属性就为0，则会被转换为false。

Saving Selections

jQuery并不会为你缓存元素。如果你做了一次选择操作，下次获取该元素时，你需要再次做同样的选择操作；你应该将选择到的元素保存到一个变量中而不是重复地做选择操作。

一旦选择的元素存储到变量中，则你可以在该变量上调用jQuery方法，就像在原始的选择上调用jQuery方法一样。

选择只会获取到 在做出选择时处于页面上的元素。如果有些元素在这之后添加到页面中，你需要重复再做选择，或者将这些要添加到页面中的元素添加到存储到变量的选择中去。那些存储到变量中的选择是不会在DOM更新的时候神奇地更新的。
